# 百诺未来业务管理系统 — 自建设计文档

## 项目概况

- **公司**：百诺未来（Biofure），6人生物试剂贸易公司，年营收~200万
- **目标**：用 AI 辅助开发（Vibecoding）自建一套业务管理系统
- **开发者**：老板本人，前端基础认知（B级），无独立项目经验
- **开发方式**：Claude Code + Next.js + AI 对话驱动
- **时间投入**：灵活，不设固定时间表
- **启动日期**：2026-05-27

---

## 技术架构

| 层级 | 选型 | 说明 |
|------|------|------|
| 框架 | Next.js 14 App Router | AI 训练数据最丰富，生成代码质量最高 |
| 数据库（开发期） | SQLite（Prisma 驱动） | 零配置，一个文件，备份=复制粘贴 |
| 数据库（上线期） | Supabase PostgreSQL | 改 Prisma 一行配置即可切换 |
| ORM | Prisma | 屏蔽 SQLite/PostgreSQL 差异，AI 最熟 |
| UI | shadcn/ui + Tailwind CSS | 表格/表单/弹窗开箱即用，视觉专业 |
| AI 工具 | Claude Code | 在 IDE 内直接对话编程，AI 驱动开发 |
| 托管（未来） | Vercel 免费版 | Next.js 官方托管，一键部署 |
| 认证（未来） | Supabase Auth | 上线时添加，不影响已有代码 |

### 关键设计决策

- **用 Server Actions 而非 API Routes**：B 级水平的开发者不需要自己定义 API 路径、处理 fetch、处理错误。一个函数完成"表单提交→写数据库→刷新页面"。
- **不做用户登录（demo 阶段）**：本地先用，后面加 Supabase Auth 不影响已有代码。
- **表名和数据用中文拼音**：业务术语本身是中文语境，降低和 AI 的沟通成本。

## 数据模型（11 张表）

### 客户模块

```
客户（Customer）
├── 公司全称、简称、统一社会信用代码
├── 类型：供应商 / 下游客户 / 两者都是
├── 行业分类：制药/检测/CRO/高校/医院
├── 分级：A/B/C
├── 客户来源：展会/网络/介绍/主动联系
├── 状态：活跃/休眠/黑名单
├── 银行账户信息
└── 备注

联系人（Contact）
├── 所属客户（外键 → Customer）
├── 姓名、电话、微信、邮箱、职务
└── 是否主要联系人
```

### 产品模块

```
产品（Product）
├── 名称（中英文）、分类、品牌、规格/货号
├── 单位（支/瓶/盒/套）、储存条件
└── 安全库存阈值

批次（Batch）
├── 所属产品（外键 → Product）
├── 批号、生产日期、有效期
├── 当前库存量
└── 供应商（外键 → Customer）

产品文件（ProductFile）
├── 所属产品（外键 → Product）
├── 文件类型：COA / MSDS / 说明书
└── 文件路径
```

### 价格模块

```
采购价（PurchasePrice）
├── 产品（外键 → Product）、供应商（外键 → Customer）
├── 单价、币种、有效期
├── 是否默认供应商
└── 变更记录（价格历史可追溯）

出货价（SellPrice）
├── 产品（外键 → Product）、客户（外键 → Customer，可为空=标准价）
├── 单价
└── 变更记录
```

### 销售模块

```
报价单（Quotation）
├── 客户（外键 → Customer）、日期、有效期
├── 状态：草稿/已发出/已确认/已过期/已转订单
└── 总金额、备注

报价明细（QuotationItem）
├── 报价单（外键 → Quotation）、产品（外键 → Product）
├── 数量、单价、小计

销售订单（Order）
├── 客户（外键 → Customer）、关联报价单（外键 → Quotation，可空）
├── 关联合同（外键 → Contract，可空）
├── 日期、状态：待确认/已确认/已发货/已完成/已取消
├── 总金额
├── 应收金额、已收金额
├── 付款方式、付款条件描述、发货条件
└── 回款日期

订单明细（OrderItem）
├── 订单（外键 → Order）、产品（外键 → Product）、批次（外键 → Batch）
├── 数量、单价、税率（默认13%）、小计
```

### 合同模块

```
合同（Contract）
├── 合同编号（如 BNWL20251103A）
├── 合同类型：采购合同 / 销售合同
├── 关联客户（外键 → Customer）
├── 签订日期、生效日期、到期日期
├── 合同总金额
├── 状态：执行中/已完成/已终止/已续签
├── 附件路径
└── 备注
```

### 关系总览

```
客户 ──┬── 联系人（一对多）
       ├── 出货价（一对多）
       ├── 采购价（一对多，作为供应商）
       ├── 报价单（一对多）
       ├── 订单（一对多）
       └── 合同（一对多）

产品 ──┬── 批次（一对多）
       ├── 产品文件（一对多）
       ├── 采购价（一对多）
       ├── 出货价（一对多）
       └── 报价明细 / 订单明细（一对多）

报价单 ──→ 订单（一对多，转换后）
合同   ──→ 订单（一对多）
```

## 界面设计

### 整体布局

经典管理后台：左侧固定侧边栏 + 右侧主内容区 + 顶部导航栏（含提醒铃铛）。

每个模块的页面结构统一：列表页（表格 + 搜索筛选）→ 详情页 → 新增/编辑对话框。

### 导航规则

| 从 | 到 | 触发方式 |
|-----|-----|---------|
| 报价单行 | 订单 | "转为订单"按钮 |
| 订单行 | 产品详情 | 点击产品名 |
| 产品详情 | 批次列表 | Tab 切换 |
| 客户详情 | 历史报价/订单 | Tab 切换 |
| 订单行 | 出库操作 | "出库"按钮（扣库存） |

### 主色调

shadcn/ui 默认中性色主题，demo 阶段不花时间调色。

## 核心业务流程

```
1. 录入客户（供应商/下游客户）
2. 录入产品 + 批次（含采购价）
3. 为客户的特定产品设置出货价
4. 创建报价单：选客户 → 加产品 → 自动带出出货价 → 生成报价
5. 报价确认 → 一键转订单
6. 订单确认 → 出库 → 自动扣库存
```

## 开发节奏

按"兴趣驱动、灵活投入"方式，拆成 11 个独立小块：

| 序号 | 小块 | 成果 | 预估对话次数 |
|------|------|------|-------------|
| 0 | 环境搭建 | `npm run dev` 能看到空白页面 | 1 |
| 1 | Prisma 建表 | 11 张表建成，数据库文件可见 | 1-2 |
| 2 | 客户列表+新增 | 搜索、筛选、新增客户 | 2-3 |
| 3 | 客户详情+联系人 | 点击客户看到详情和联系人 | 1-2 |
| 4 | 产品列表+批次 | 产品管理 + 批次追踪 | 2-3 |
| 5 | 采购价+出货价 | 价格关联到产品/客户 | 2-3 |
| 6 | 报价单生成 | 选客户→选产品→生成报价 | 2-3 |
| 7 | 报价转订单 | 一键转订单 + 订单管理 | 2-3 |
| 8 | 出库+库存扣减 | 订单出库 + 批次库存变化 | 2-3 |
| 9 | 合同管理（轻量） | 合同列表+上传+关联订单 | 1-2 |
| 10 | 首页仪表盘 | 统计卡片（客户数/订单数/待回款） | 1-2 |

每次坐下来完成一小块，小块间独立性强。

## 迁移计划（未来）

开发完成后从 SQLite 迁移到 Supabase PostgreSQL：

1. 改 Prisma 配置：`datasource db { provider = "postgresql" }`
2. 改连接字符串为 Supabase URL
3. `npx prisma db push` 一键建表
4. AI 辅助写数据迁移脚本
5. 部署到 Vercel

预估迁移工作量：1-2 小时。
